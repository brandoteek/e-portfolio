Secure Software Development

Secure Software Development Module

Learning Outcomes

- Develop an understanding of the basic principles of secure development methodologies.
- Explore how to undertake analysis, program design, software construction and testing required for software development.
- Demonstrate an understanding of the basic principles of architecture, as well as traditional and contemporary Software Development Life Cycle (SDLC) models, such as TOGAF and Agile.

REFERENCES:
- Core Text: Buelta, J. (2022) Python Architecture Patterns: Master Api Design, Event-Driven Structures, and Package Management in Python. Birmingham, UK. Packt Publishing Ltd.
- Sharma, A. & Bawa, R. K. (2020) Identification and Integration of Security Activities for Secure Agile Development. International Journal of Information Technology.
- Lecture casts, readings and Module information from the University of Essex portal.

UNIT 1 - Introduction to Secure Software Development

- Explore the waterfall and agile approaches to software development, with a focus on the implications of developing
secure software using each. - Become acquainted with the Unified Modelling Language and how it can be used to support software development.
- Gain familiarity with the standards which are used by industry to develop secure software.
- Appreciate the importance of developing a risk-aware culture within an organisation.

WIKI - Keywords:
- OWASP 'broken access control'
- OWASP 'Coding weaknesses'
- Cross-Site Scripting (XSS) Vulnerabilites
- Visual Paradigm.
- Sequence Diagram .
- Umbrello

UNIT 2 - UML Modelling to Support Secure System Planning

- Awareness of the opportunities for integrating software security development practices at each stage of the agile SDLC.
- Understand the range of ways in which software security can be problematic.

WIKI - Keywords:
- Identification and Integration of Security Activities for Secure Agile Development
- Unified Architecture Framework (UAF) Domain Metamodel.

UNIT 3 - Introduction to Programming Languages

- History and concepts of programming languages.
- Investigate language concepts using Python as an example.
- Best practices and methods to overcome common security issues.
- Key milestones in the development of programming languages.
- Key paradigms that define the different types of languages.
- Key concepts that determine the operation of programming languages.
- Key programming challenges and recommended best practices.
- Design patterns are and when to use them.

WIKI - Keywords:
- Buffer Overflow - What factors determine whether a programming language is secure or not? - Could Python be classed as a secure language? - Python would be a better language to create operating systems than C.

UNIT 4 -Introduction to Testing

- Explore the range of questions to ask when developing a test plan.
- Become familiar with two industry software testing standards.
- Examine a range of testing techniques and appreciate the relevance of each in different testing scenarios.
- Tools and frameworks which are available to support and automate the Python testing process.
- Reflect on the utility of an ontology to support a distributed architecture.

WIKI - Keywords:
- Linters
- Flake8
- pylint
- Cyclomatic Complexity - mccabe

UNIT 5 - Future Trends in Secure Software Development

- Discuss future trends in the areas of secure design, development, programming languages and operating systems.
- Explain what is meant by Fog Computing, the Internet of Things and Cyber Physical Systems.
- Explore systems and solutions that can be investigated as possible research topics or fields.

WIKI - Keywords:
- IoT, Cyber Physical Systems, Blockchain

UNIT 6 - The Security of Programming Languages Debate

- Review the security capabilities of a selection of programming languages, which include F#, Python, Rust, and Swift.
- Engage in a debate about the above languages.
- Explore some of the advances needed to support secure programming.

WIKI - Keywords:
- F#, Rust, Swift
- Functional Programming.

Introduction
The current reﬂective essay follows Gibbs reﬂective cycle (University of Edinburgh, 2024). The essay is divided in several sections to point out the experience obtained from the Secure Software Development module.

Description
Initially the module challenges us with the creation of a design of a Music application that can perform Create, Update, Read, Delete operations (CRUD).
While interacting with my peers, everyone came with diKerent level of knowledge regarding software engineering and databases.
The design phase oKers us an opportunity to contribute in the diKerent parts of the project, like the UML diagrams, Database design relationships, and the inter-relationship between these diagrams and components.
Due to time zones of several of us in the team, it was not simple to agree in speciﬁc meetings. Therefore, the use of communication tools like WhatsApp and specially MS Teams provided by the University was quite useful for the coordination.
The biggest challenge came with the Individual coding activity, since myself I have a background in network engineering, it has been one of the most challenging responsibilities due to my limited language programming experience.

Feelings
I could notice that everyone in the Team wanted to contribute, even myself although my programming level. In the design phase everyone shared their part, and I was proud to share time and energy with a motivated team.
However, I struggled to connect properly the elements from the design into real programming language, like Python.
I never felt in control while programming the design. Plenty of frustration happened but I tried as much as I could to provide something.
Procrastination sabotaged my progress several times. I experienced that with programming, but to be fair I tried to overcome that feeling and only through practicing little by little I could see that I started to understand progressively. Little victories that feel a diKerence at long term. I am trying to improve my patience level while learning programming by continuously having this in mind.
I am aware that without trying is impossible to improve, so I nowadays I spend hours trying to comprehend a piece of code. I found it admirable that some people can understand the complexities of translating an idea, an algorithm into a computer language, but I understand that some of them had to overcome their own procrastination issues and took action to obtain the level in programming they have today.

Evaluation
While working collaboratively, we have respect to each other but at the same time we were not directly confrontative in points that could be improve our design. We tend to avoid “hurting” any feeling or “judgement” to avoid, also disagreements. This could have limited our ﬁnal design and diagrams. However, while working in a team it’s possible to see oneself weakness and areas for improvement, due to the group questions and liabilities.
It was challenging to implement and comprehend the interaction between the artifacts in Python language. It demanded Object Oriented Programming knowledge and information correlation.

Analysis
Teamwork is not simple when everyone has an opinion and perspective. However, my team was professional and respectful. Everyone listens to every idea and opinion, which made it comfortable and not tedious. Additionally, regarding the module, it was interesting to get familiarized with OWASP, software development practices, and learn about automated security tools like Pylint and security testing tool like Bandit. To refactor and reﬂect about the written code and have a deeper understanding of the created script.

Conclusion
Even though, I recognize my diKiculties to interact with a programming language, I started with baby steps, repeating same lines of codes until is necessary, and in this resubmission even though is not the advanced level I would like to provide. Personally, I felt that the perseverance paid oK. I felt more conﬁdent while using functions, operators, iterations, although there is still plenty to learn and absorb. Action Plan
I decided to perfectionate my Python skills and taking it to the next level but continue practicing time to time, like a normal habit, part of the daily habits. As every skill, is the only way to achieve the desired level.
